# B站搜索工作流

适用场景：
- 用户明确提到 `B站` / `哔哩哔哩` / `bilibili`
- 用户要求 `搜索`、`整理视频`、`列出最火/最热视频`

## 目标

在 Electron 右侧浏览器区域内，用 **同一个站内搜索结果页或列表页** 完成任务。

## 强制流程

1. 不要先自由点击首页元素。
2. 不要先进入 UP 主空间页、独立播放页、外部搜索引擎。
3. 优先直接构造并打开 B 站站内搜索 URL。
4. 如果需要关键词编码，优先调用跨平台工具 `node tools/build_platform_search_url.mjs bilibili "关键词"` 生成 URL；Linux/macOS 也可用 `tools/build_platform_search_url.sh bilibili "关键词"`。
5. 得到 URL 后，必须用 `browser.navigate({ url })` 在当前 Electron 浏览器区域打开。
6. 打开后先 `browser.getState()` + `browser.screenshot()` 确认还在 `bilibili.com`。
7. 如果任务是“整理最火的十个视频”，优先停留在：
   - B站搜索结果页
   - B站频道/视频列表页
   - B站排序页
8. 除非用户明确要求打开某个具体视频，否则不要点击会进入独立播放页的视频标题。
9. 除非用户明确要求进入 UP 主主页，否则不要点击头像、昵称、空间入口。
10. 如果页面误跳到错误路径，立刻重新 `browser.navigate` 回搜索结果页，不要继续。

## 推荐 URL 模式

站内搜索：

```text
https://search.bilibili.com/all?keyword=<编码关键词>
```

## 整理视频时的原则

1. 先把结果稳定停在列表页。
2. 优先从当前列表页提取前 10 个视频卡片信息。
3. 如果需要排序，尽量点当前搜索页里的排序按钮，不要离开到别的平台。
4. 如果结果页里混有专栏/用户/直播，优先只筛视频卡片。

## 禁止事项

- 禁止跳到 YouTube/Google/Bing 找替代结果
- 禁止把“搜索任务”变成“随便打开一个视频看看”
- 禁止把“整理十个视频”变成“进入作者主页后再慢慢找”
