# HTML Game Generation

适用场景：用户要求“生成一个 HTML 游戏 / 网页并打开”。

## 布局要求

1. 页面必须针对 Electron BrowserView 设计，优先铺满可见区域，不要只在中间放一个小画布
2. 避免大面积纯黑边、纯空白边；背景和游戏区域要形成完整画面
3. `canvas` 或主游戏容器必须自适应：
   - `width: min(96vw, ...)`
   - `max-height: 94vh`
   - 高度跟随宽度缩放
4. HUD/状态栏不要挡住主要操作区域
5. 主游戏区域验收标准：
   - 宽度至少接近 BrowserView 宽度的 70%
   - 高度至少接近 BrowserView 高度的 60%
   - 不要使用“800x600 固定尺寸 + body 居中 + 黑底”这种偷懒布局

## 交互要求

1. 打开页面后不要自动进入失败状态
2. 优先做“开始页 / 按键开始 / 点击开始”，不要一加载就自动跑逻辑
3. 如果有生命、敌人、倒计时，初始几秒不要让玩家站着就立即死亡
4. 如果角色会重生，给短暂无敌时间或安全出生点

## 交付要求

1. 文件写到 `agent/` 目录
2. 生成完成后必须 `browser.navigate(file://绝对路径)` 打开
3. 打开后必须 `browser.screenshot()` 自检
4. 如果截图里出现以下任一情况，必须继续修正后再结束：
   - 游戏区域过小
   - 四周黑边或空白边过大
   - 一打开就“游戏结束 / 失败 / 死亡”
   - 文字或按钮明显遮挡主画面
5. 自检时不要只看文件有没有打开，要看视觉结果是否像一个完整可玩的页面

## 默认实现建议

1. 用单文件 HTML（内联 CSS/JS）
2. 默认做响应式容器，而不是写死 800x500 后只靠 body 居中
3. 如果是平台跳跃/动作类游戏，默认提供：
   - 开始覆盖层
   - 重开按钮
   - 暂停或未开始状态
4. 配色优先明亮、完整，不要默认黑底小框
